# План: PRD -> реализация

Собираем минималистичный статический сайт на чистом HTML/CSS/JS. Реальные URL без hash-роутинга: `/`, `/all/`, `/credits/`, `/works/{slug}.html`. Структура в `docs/` (GitHub Pages), с отдельным реестром работ и папкой модулей. Навигация и список работ строятся из реестра. Каждый модуль — отдельный файл с `init()`/`start()`/`destroy()` и корректной очисткой ресурсов.

## Шаги
1. Зафиксировать деплойную структуру в `docs/`: `index.html`, `styles.css`, `app.js`, `registry.js`, `works/`, `all/index.html`, `credits/index.html`, `works/{slug}.html`.
2. Спроектировать shell-страницы: контейнер для арта, текстовая навигация, места под `credits`/`all posts`, без кнопок и без скролла; отступы по PRD.
3. Реализовать реестр работ в `docs/registry.js` с полями `slug`, `title`, `order`, `script`, `meta` и экспортом массива для навигации и списка работ.
4. Сгенерировать статические страницы по шаблону:
	- `/all/index.html` — индексируемый список ссылок на `/works/{slug}.html`.
	- `/credits/index.html` — индексируемая текстовая страница.
	- `/works/{slug}.html` — отдельная страница работы с `data-slug` или мета-вставкой для загрузки нужного модуля.
5. Реализовать клиентскую навигацию через History API (без hash) в `docs/app.js`: переходы по ссылкам, `next`/`random` по правилам PRD.
6. Добавить загрузчик модулей и lifecycle-управление: монтирование в общий контейнер, запуск `init()`/`start()`, корректный `destroy()` при переключении страниц и работ.
7. Создать базовый пример модуля в `docs/works/` для проверки жизненного цикла и очистки ресурсов.
8. Настроить стили для минималистичного интерфейса и фиксированного холста без скролла в `docs/styles.css`.

## Проверка
- Ручная проверка в браузере: `/` автозапуск, `next` и `random` работают, `random` не выбирает текущую.
- `/all/` показывает индексируемый список ссылок, `/credits/` показывает текстовую страницу.
- Прямой заход на `/works/{slug}.html` открывает нужную работу без промежуточного роутинга.
- При переходе между страницами и работами не остаются активные таймеры/анимации.

## Зафиксированные решения
- Реальные URL без hash; переходы через History API.
- Структура в `docs/`: `index.html`, `styles.css`, `app.js`, `registry.js`, `works/`, `all/`, `credits/`.
- Метаданные хранятся только в реестре и не отображаются в UI в MVP.
