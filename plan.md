# План: PRD -> реализация

Собираем минималистичный статический сайт на чистом HTML/CSS/JS. Одна shell-страница с hash-роутингом для `all posts` и `credits`. Структура в `public/`, с отдельным реестром работ и папкой модулей. Навигация и список работ строятся из реестра. Каждый модуль — отдельный файл с `init()`/`start()`/`destroy()` и корректной очисткой ресурсов.

## Шаги
1. Создать базовую структуру файлов в `public/`: `index.html`, `styles.css`, `registry.js`, `works/`.
2. Спроектировать shell-страницу: контейнер для арта, текстовая навигация, места под `credits`/`all posts`, без кнопок и без скролла; отступы по PRD.
3. Реализовать реестр работ в `public/registry.js` с полями `slug`, `title`, `order`, `script`, `meta` и экспортом массива для навигации и списка работ.
4. Реализовать hash-роутинг в `public/index.html` и `public/registry.js`: `#/` для работы, `#/all`, `#/credits`, `#/work/{slug}` для deep link; `next`/`random` по правилам PRD.
5. Добавить загрузчик модулей и lifecycle-управление: монтирование в общий контейнер, запуск `init()`/`start()`, корректный `destroy()` при переключении.
6. Создать базовый пример модуля в `public/works/` для проверки жизненного цикла и очистки ресурсов.
7. Настроить стили для минималистичного интерфейса и фиксированного холста без скролла в `public/styles.css`.

## Проверка
- Ручная проверка в браузере: `#/` автозапуск, `next` и `random` работают, `random` не выбирает текущую.
- `#/all` показывает список ссылок, `#/credits` показывает текстовую страницу.
- При переходе между работами не остаются активные таймеры/анимации.

## Зафиксированные решения
- Роутинг через hash (`#/all`, `#/credits`).
- Структура в `public/`: `index.html`, `styles.css`, `registry.js`, `works/`.
- Метаданные хранятся только в реестре и не отображаются в UI в MVP.
