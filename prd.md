# PRD  
## Generative Art Website

---

# 1. Цель проекта

Создать минималистичный веб-сайт для публикации generative art работ.

Основные принципы:

- один экран — одна работа
- работа **автоматически запускается**
- максимально **минималистичный интерфейс**
- новые работы добавляются **простым добавлением файлов**
- сайт хранится и обновляется **через GitHub**

Сайт должен ощущаться как **арт-журнал или лаборатория**, а не как галерея.

---

# 2. Основной UX

## Главная страница

При открытии сайта:

- загружается **одна generative art работа**
- она **сразу запускается**
- никаких кнопок play
- никакого скролла
- вся работа занимает экран часть экрана (весь экран  минус отступы на шапку, и подвал по 10% и слева\справа 5%). для всех работ один канвас

UI должен быть минимальным.

---

## Навигация

На странице присутствуют:

### правый верхний угол

credits
all posts


### правый нижний угол

next
random
deep link на текущую работу


просто текст, никаких кнопок

### Функции

**next**

- открывает следующую работу

**random**

- открывает случайную работу
- не должен выбирать текущую

---

## Страница "All Posts"

Отдельная страница со списком всех работ.

Формат списка:

work title
work title
work title


Без:

- превью
- картинок
- карточек

Просто список ссылок.

---

## Страница "Credits"

Минимальная текстовая страница.

Содержит:

- автор
- технологии
- контакты
- ссылки

---

# 3. Контентная модель

Каждая generative работа считается **постом**.

Пост содержит:

| поле | описание |
|-----|------|
slug | уникальный id |
title | название |
order | порядок |
script | файл с логикой |
optional meta | дата размещения, описание, ссылки |

---

# 4. Архитектура сайта

Сайт реализуется без фреймворков.

Основная идея:

**одна оболочка + список работ + отдельные арт-модули**

---

## Основные компоненты

### 1. Shell страницы

Главная HTML страница содержит:

- контейнер для арта
- минимальный UI
- навигацию

Она **не содержит конкретных работ**.

---

### 2. Реестр работ

Один файл со списком всех работ.

Назначение:

- список постов
- порядок
- данные для навигации
- источник для страницы `all posts`

---

### 3. Арт-модули

Каждая generative работа:

- отдельный файл
- содержит JS код
- создаёт canvas / WebGL / Three.js
- запускается автоматически

---

### 4. Контейнер арта

Shell предоставляет DOM контейнер.

Каждая работа:

- монтируется в контейнер
- запускает анимацию
- при переходе уничтожается

---

# 5. Навигационная логика

## Next

Алгоритм:

1. получить текущий индекс
2. взять следующий
3. если конец списка → перейти к первому

---

## Random

Алгоритм:

1. выбрать случайный индекс
2. если совпадает с текущим → выбрать заново

---

# 6. Жизненный цикл работы

Каждая работа должна поддерживать:

### init()

создание:

- canvas
- WebGL
- сцены
- таймеров

---

### start()

запуск анимации

---

### destroy()

очистка:

- requestAnimationFrame
- event listeners
- WebGL context
- DOM

Это нужно чтобы при переключении работ **не накапливались ресурсы**.

---

# 7. Добавление новой работы

Процесс добавления новой работы должен быть максимально простым.

Шаги:

1. создать новый файл работы
2. добавить запись в реестр работ
3. push в GitHub

После деплоя работа:

- появляется в `next`
- появляется в `random`
- появляется в `all posts`

---

# 8. Технические требования

## Хостинг

GitHub Pages.

Сайт должен быть:

- полностью статическим
- без сервера
- без backend

---

## Поддерживаемые технологии для арта

Допускается:

- Canvas 2D
- WebGL
- Three.js
- GLSL shaders

---

## Производительность

Работы должны:

- запускаться быстро
- не блокировать UI
- корректно освобождать ресурсы при переходе

---

## Адаптивность

Минимальные требования:

- корректная работа на desktop
- корректное масштабирование canvas

Mobile поддержка — optional.

---

# 9. Дизайн

Минимализм.

Рекомендуемые принципы:

- темно серый фон
- максимум экранного пространства для арта
- маленькая типографика
- без UI-шума

Сайт должен ощущаться как **интерактивная выставка**.

---

# 10. Будущие расширения (не обязательно)

Потенциальные функции:

- seed sharing
- keyboard navigation
- fullscreen mode
- режим серии работ
- tags / categories
- generative presets

Эти функции не входят в MVP.

---

# 11. MVP Scope

Для первой версии требуется только:

- одна оболочка сайта
- загрузка работ
- next
- random
- all posts
- credits
- автозапуск работ

---

# Итог

Получается сайт со следующими свойствами:

- полностью статический
- хостится на GitHub Pages
- минимальный UI
- одна generative сцена на экран
- работы добавляются как файлы
- навигация строится из списка работ

По сути это **микро-движок generative art сайта**, но без фреймворков.
